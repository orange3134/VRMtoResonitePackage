# Release

Releases are published from the local machine because the build needs local
Resonite DLL references.

Prerequisites:

- Resonite is installed locally, or pass `-ResonitePath`.
- GitHub CLI is installed.
- `gh auth login` has already been completed.

Build and upload `publish\VrmToResonitePackage.exe` to a GitHub Release. The
release tag is derived from the version embedded in the built EXE, for example
`v2026.06.14.1234`:

```powershell
.\release.ps1
```

Create and push the tag first, then create/upload the release:

```powershell
.\release.ps1 -CreateTag -PushTag
```

Use a non-default Resonite install path:

```powershell
.\release.ps1 -ResonitePath "D:\Games\Resonite"
```

Override the automatically derived tag only when you need to publish a specific
manual tag:

```powershell
.\release.ps1 -Tag v2026.06.14.1234
```

Only `VrmToResonitePackage.exe` is uploaded. The PDB generated by
`publish.ps1` remains local.
